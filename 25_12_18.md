# Github로 협업하기
## .gitignore
### 1. .gitingnore 이란?
> 특정 파일 혹은 폴더에 대해 Git이 버전 관리를 하지 못하도록 지정하는 것

- .gitignore를 하는 이유
  - 불필요한 파일이 커밋되는 것을 막기 위해
  - 개인 환경 차이로 인한 충돌을 줄이기 위해
  - 민감정보가 실수로 올라가는 걸 줄이기 위해
  - 저장소를 가볍고 재현 가능하게 하기 위해


## clone , pull
### 1. clone
> 원격저장소 자체를 복제

`git clone <원격 저장소 주소> <복제한 레포지토리 폴더이름 지정>`의 형태로 작성합니다.

처음에 한 번만 하면 됩니다


### 2. pull
> 원격저장소의 변경사항을 가져와서, 로컬저장소를 업데이트

`git pull origin main` 업데이트된 내용을 가져옵니다.

### 3. push
> 업데이트된 내용을 다시 업로드 하기 위해 사용합니다.

`git push origin main` 

- 참고
origin은 일종의 도로명이라고 생각하면 됩니다.
도로명을 origin이라고 하는 것은 일종의 관례입니다.
